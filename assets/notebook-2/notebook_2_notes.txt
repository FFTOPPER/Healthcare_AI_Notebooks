Data Access from PostgreSQL, Google Drive, and Supabase - Notes
===============================================================

Project Objective
-----------------
This notebook demonstrates how to access and compare data from three
commonly used data sources:

1. PostgreSQL database hosted on Neon
2. CSV file stored in Google Drive
3. Supabase cloud backend

The goal is to show how structured data can be retrieved, validated,
and benchmarked across different storage platforms.

Dataset Details
---------------
Dataset Name     : Synthetic E-Commerce Orders
Total Records    : 100
Total Features   : 11

Columns
-------
- order_id
- customer_name
- product_name
- category
- quantity
- price
- order_date
- payment_method
- city
- order_status
- rating

Technologies Used
-----------------
- Python
- Pandas
- SQLAlchemy
- psycopg2
- Google Colab
- Google Drive
- Supabase Python Client
- Neon PostgreSQL

Data Sources
------------
1. PostgreSQL (Neon)
   - Cloud-hosted relational database
   - Queried using SQLAlchemy and psycopg2

2. Google Drive
   - CSV file storage
   - Loaded directly using Pandas

3. Supabase
   - Backend-as-a-Service
   - PostgreSQL-powered REST API

Validation Results
------------------
- Google Drive returned 100 rows and 11 columns.
- PostgreSQL returned 10 sample rows and 11 columns.
- Supabase returned 10 sample rows and 11 columns.
- Column structures were consistent across all sources.

Performance Benchmark
---------------------
Data retrieval time was measured for each source.

Example Results:
- PostgreSQL : ~7.24 seconds
- Google Drive : ~0.006 seconds
- Supabase : ~0.77 seconds

Interpretation:
- Google Drive was fastest because the CSV was already available locally.
- Supabase was faster than direct PostgreSQL in this notebook.
- PostgreSQL showed network overhead during connection establishment.
